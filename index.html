关于“小程序”的几点思考

2018-07-10
  1. 小程序更接近Web而不是App，是一个个离散的页面，不能用应用的视角去思考问题。
  2. 小程序的互动性和传播性能很好地解决被动性需求；主动性需求如何更好地解决是微信接下来关注的重点。
  3. 微信越来越像操作系统，值得考虑的是下一代操作系统，社交关系、支付等基础能力是否会从应用级别下沉到系统级别。
  4. 小程序与用户的连接性没有很好地解决，目前服务通知是一种暂时性的处理办法。
  5. 小程序应用场景进入下一层级后，会越来越少强调“用完即走”这件事。
  6. 超链接是互联网最伟大的发明之一，有可能换一种形式在小程序内复现，从而打破目前信息和服务的封闭状态。

缅怀“用音乐倾听彼此”

2018-07-10

上学时最经常上的网站有两个，一个是SongTaste（用音乐倾听彼此），另一个是有意思吧。前者因为版权问题，从可以下载变成只能在线收听，后来索性被关站，现在已经无法访问了；后者工作后看的慢慢就少了，今天想起来看一下的时候，发现主页已经变成“江湖仍在，我们再见！”，不禁有些唏嘘。

有一类人听歌从来不看歌名和歌手，比如我。所以听过的歌无数，能想起来的几乎没有。这种随便听听的个性和SongTaste的产品机制简直绝配。SongTaste上的博主都是人才，不仅从海量的歌中帮你把好听的歌挑出来（那时候做歌单远没有现在这么方便），还偶尔会配上一段文字，看得出有很多专业的人在很用心地维护内容。虽然小众，但整个产品社区的氛围还是相当好。网易云音乐简直就是它的降级版本。

“有意思吧”的产品模式更简单，就是一个简单的PGC博客，每天都会发一些让你会心一笑的文章，以及挑一些好听的歌曲集合。截止关站之前，一直都保持着自己的调性不温不火地发展着。唉，结果这种人畜无害的产品居然也关站了。

互联网发展越来越快，当一切都标准化，快餐化，也就越来越没意思了。想起罗素那句“须知参差多态，乃是幸福本源”。

由产品“多抓鱼”想到的

2018-05-31

“多抓鱼”是一个二手书交易平台，是一个让人看一眼就会喜欢上的产品。也是我理想的创业产品范例，它模式足够简单但巧妙、可控、能持续，有一群可爱的用户。

早期做二手书的产品有很多，既有做C2C的，也有做针对大学生群体B2C的，但成气候的几乎没有。究其原因，可能是把二手书仅仅当成了二手书的生意。“多抓鱼”充分地利用中间地带的特质，利用科学手段透明定价，统一收购，从源头按需控制库存，有意培养平台的品牌气质，让用户在买书卖书的过程中接触的介质变成似乎存在的有形实体。这就是它成功的地方。

“多抓鱼”看似在做二手书的生意，实质上是在做社区。反过来说也成立，社区的本质是通过“一件东西”把一批人撮合起来，对“多抓鱼”而言，这件东西就是二手书。

当然，它一定还是个小众的生意，但好玩，有意义。有了品牌价值后，也有了更多可玩的空间。祝福“多抓鱼”。

尝到甜头了并非好事

2018-05-15

赌博、酗酒为何容易让人深陷其中？抖音为何有让人沉迷的魔力？为什么买了一套房的人会拼命想买第二套第三套？为何区块链一夕之间火遍大江南北？

答案就是：”他们尝到甜头了！“

事情要从年初说起。我一个亲戚突然找到我，说：”你知道区块链吗？我正在做一个区块链的项目，各方面都完备了，就差把产品弄出来…….“，他没继续往下说，我也八八九九猜到他要做什么。无非是把发展下线返佣这一套换了个区块链的壳，弄些虚拟货币让大爷大妈们买。

发币的技术门槛早已经被前辈们趟平了，剩下就是比忽悠能力、渠道能力。所以这一年区块链创业者蜂拥而至，在没有清晰的落地场景下，怀揣着币圈一夜暴富的梦，不仅给用户灌迷魂汤，有些人狠起来，连自己都灌。

所有人都会说“我觉得数字货币本身是泡沫，但区块链技术还是有价值的”，我也部分同意。可难就难在有价值的地方在哪里？区块链技术能恰好解决新的业务场景吗？区块链技术能更好地改造老的业务场景吗？这里面并不是简单拍脑袋就能想清楚的。

这也不怪创业者，比特币近几年的表现，让无数人扼守叹息。觉得自己错过了一个个成为亿万富翁的机会。所以暂时看不懂不重要，先上车最重要，万一成了呢！不仅没上车的如此，赚到钱的更是如此，一旦尝到了与付出不成正比的财富回报方式，再想回到兢兢业业的状态就太难了。

所谓的“尝到了甜头”，正是如此。偶然间得到的获取超额回报的方法，会让人越来越狭隘和盲从。那种不信邪的精神，不真正栽个大跟头是不会回头的。

所以，赌博是尝到了一把天下我有的甜头；酗酒是尝到了10元钱进入精神乐园的甜头；沉迷抖音是尝到了免费感官刺激的甜头；拼命买房是尝到了房价上涨的甜头；区块链爆火是尝到了一夕暴富的甜头。

越是拼命迷恋这些甜头，社会就越投机，我不希望这样。

古代中国居然存在《西部世界》中的人造人？

2018-05-12

《列子·汤问》里记载了这么一则寓言故事。

周穆王西出巡狩，越过昆仑，登上弇（yan）山。在返回途中，有一位叫偃师的工匠向穆王献技。王召见之，问道：”你有什么能耐？“偃师回说：”只要是天子您的要求，我都可以尝试。可是臣已经制造了一个东西，希望您先看看。“穆王说：”明天你把它带来，我和你一起观之。“

第二天，偃师按约定来拜见穆王。王召见之，问：”和你一起来的是谁呐？“回答说：”是臣制作的能跳舞的倡者。“穆王吃惊地打量它，走路俯仰，和真人无异。

真是巧妙啊！倡者仰起头颅，就合律而歌；抬起手臂，就应节而舞。千变万化，随心所欲。穆王认为这是真人，叫来盛姬和侍妾一起观看。

舞蹈即将结束时，倡者向穆王左右的侍妾眨眼挑逗。穆王大怒，认为偃师拿真人戏弄自己，要立刻杀了他。偃师十分恐惧，立刻将倡者剖开给穆王看，都是皮革、木头、凝胶、漆以及白色、黑色、红色、青色物体。穆王仔细查看，发现倡者里面有肝胆、心肺、脾肾、肠胃，外面有筋骨、关节、皮毛、牙齿和头发，都是假物，人身上的倡者身上无一不有。把他们合到一起后又像刚开始见到那样。

穆王试着取下它的心脏，则它口不能言；取下肝脏，则目不能视；取下肾脏，则足不能步。穆王这才高兴地叹道：”人的技艺竟能与造化者同功！“于是他下令用副车载着倡者回去了。

十分好奇列子是如何突发奇想写出这么一则寓言的，倡者对穆王侍妾的挑逗让整篇文章不寒而栗。穆王载着倡者而归，又是否达到了偃师的目的？这里面不妨开一下脑洞：

周穆王时，武王灭商不过50来年，商朝”余孽“未必全灭，偃师献技可能怀有二心。首先，在现在洛阳市偃师发现最早的商朝遗址，可能是商朝最早的都城；其二，商朝是和其后的诸多王朝有非常大的不同，商朝迁都频繁，被其他部族称之为天邑城；其三，商朝很可能科技发达，甚至远远超过现在。我们都知道封神榜里周商两派各自用法宝斗来斗去，实际上高级的科技看起来确实像魔法，所以被后人包装臆会；

那后来为何周能灭商？而后来的科技为何又回归到刀耕火种的时代？第一个问题，可能是周人也掌握着这种技能或者商人出了叛徒；第二个问题，可能是前朝文化一律抹杀，也可能是双方一场核战一切归零。

这就不难理解为何偃师有这样高超的技艺，并且献技于穆王了。

引《偃师献技》原文：

周穆王西巡狩，越昆仑，不至弇山。反还，未及中国，道有献工人名偃师。穆王荐之，问曰：“若有何能？”偃师曰：“臣唯命所试。然臣已有所造，愿王先观之。”穆王曰：“日以俱来，吾与若俱观之。”翌日偃师谒见王。王荐之，曰：“若与偕来者何人邪？”对曰：“臣之所造能倡者。”穆王惊视之，趋步俯仰，信人也。巧夫！领其颅，则歌合律；捧其手，则舞应节。千变万化，惟意所适。王以为实人也，与盛姬内御并观之。技将终，倡者瞬其目而招王之左右侍妾。王大怒，立欲诛偃师。偃师大慑，立剖散倡者以示王，皆傅会革、木、胶、漆、白、黑、丹、青之所为。王谛料之，内则肝胆、心肺、脾肾、肠胃，外则筋骨、支节、皮毛、齿发，皆假物也，而无不毕具者。合会复如初见。王试废其心，则口不能言；废其肝，则目不能视；废其肾，则足不能步。穆王始悦而叹曰：“人之巧乃可与造化者同功乎？”诏贰车载之以归。

Unicode 和 UTF-8

2018-01-12

最近在写程序的时候遇到了一些编码问题，做了一些研究，简单记录一下。

Mac OS 默认安装的Python版本是2.7，我也懒得折腾，一直基于2.7版本工作。但是前两天在处理Emoji的时候，发现用Python遍历字符串中的字符时，会默认把单个Emoji表情拆分成多个字符。

产生这个问题的原因是Python2.7版本处理Unicode字符是基于USC2编码，即使用两个字节来标识单个Unicode字符，但是部分Emoji表情已经超出USC2的范围，应该使用USC4来处理。

解决这个问题的办法有两种：

  1. 重新编译Python2.7，并指定Unicode编码方式为USC4
  2. 使用Python3.3+，默认使用USC4

我决定使用第二种方式，因此，我也打算将应用版本升级到最新的Python3.6.4。在研究这个问题的过程中，也对Unicode字符集有个更深的了解。

Unicode 顾名思义，是为了解决不同语言和符号在数字世界统一显示的问题，是使用“U+”开头的十六进制的数字串。被称为“Code Point”。

全世界所有的字符假如超过10万个，那么就需要至少3个字节来表示一个“Code Point”，2个字节16比特最大表示65536。并且Unicode是可以扩充的字符集，并没有限制字符个数。

如果直接使用“Code Point”来映射字符，那么Unicode字符集扩展进行扩展时，到底使用几个字节来表示一个字符就会变成一件不确定的事。另外还有一件不能忍受的事是在表示英文字母和数字时（互联网上绝大部分内容），本来使用一个字节就能搞定的事变成使用四个字节（假定），带来了巨大的存储和网络带宽浪费。

于是对Unicode编码就变得很有必要了，要让计算机理解Unicode数字串究竟代表那个字符，就必须告诉计算机编码标准，目前常见的编码方式有“ASCII”、“UTF-8”、“UTF-16”等，旧时代遗留下来的上百种编码方式，也可以运用到对Unicode的字符集的编码上，于是皆大欢喜。

其中最流行的编码方式是“UTF-8”，这是一种可变长的编码方式。换句话说，如果是英文字符，采用“UTF-8”编码，则占用一个字节；如果是中文，采用“UTF-8”编码，则占用二个字节。所以采用UTF-8编码的文件，单个字符占用的字节数从1个到多个不等。

所以在Python2.7判断字符串长度时，无法识别超过2个字节的字符，所以直接拆分处理了。

上文只是基于这两天研究的一个总结，部分细节可能不准确，欢迎勘正。

评《自由选择》

2017-12-13

《自由选择》是大名鼎鼎的自由派经济学家弗里德曼的著作。看这本书之前我并不认识他，因为我对经济学一无所知，正因为无知，才激发了求知的好奇。整本书看完，说不上特别理解，但有一些观点是可以回忆起来的。

  1. 推崇市场这个看不见的手，强调价格的信息传递作用。政府的职责范围越来越大，导致越来越多的行政政策使价格传递的信息失效，从而使市场变得盲目。
  2. 工会的职责之一是建立准入门槛，保证服务提供者的稀缺性，导致了服务价格高昂、服务供给不足。这一点在医疗行业表现得尤其明显。开放管制，让大众有自由选择职业的权利是治疗这些疾病的良方，让消费者用脚投票。
  3. 扼制垄断最好的手段是充分竞争，只要国家间充分自由贸易，就不会产生垄断。
  4. 政府为了解决问题，不断成立新的部门，导致利益群体扩大，行政成本逐渐大于获得的收益。解决问题的良方依然是充分相信市场的力量。
  5. 弗里德曼应该是非常推崇亚当·斯密的，在书中多次引用亚当·斯密的观点。
  6. 认为美国现行的教育制度，表面上是补贴穷人，实际上是剥削穷人，补贴富人，加剧了不公平。

在书的最后，弗里德曼针对美国当时的情况，提出了几条建设性的修正案，摘抄几点比较有意思的如下：

  1. 国会不得制定任何法律限制商品和劳务供应者对其商品和服务进行定价的自由。
  2. 各州不得制定和执行任何法律，使得合众国的公民从事其自选工作或职业的自由受到限制。
  3. 国会有权就个人收入进行课税，且不对收入来源进行区分，所得税收不与其他各州进行分配，也不必参考人口普查或统计资料，所得税起征点中不包括规定数额的职业费用、商业费用和个人津贴，对于超过起征点的个人所得实行统一的税率。所谓“个人”，不包括公司和其他类型的法人。
  4. 所有与美国政府和其他方面签订的，以美元计价货币的合同，以及联邦法律中规定的美元总额，每年都应依据前一年的一般物价水平进行调整。

虽然是比较理想化的建议，但弗里德曼的思想可见一斑。

下面简单说下个人体会。书里有些观点不免偏激，也有时代的局限性。中国是一个市场加宏观调控的国家，甚至宏观调控还多一点。从结果来看，虽然面临很多问题，但总体是非常成功的。俗话说，实践是检验真理的唯一标准。自由市场是不是万能药，如果是自由市场中国能不能发展的比现在好，谁都不敢说。

通向真理的路往往不止一条，一切只能待时间来证明。

好格式，坏文章

2017-09-30

对于入行不久的交互设计师而言，甚至是有些资深设计师，我常常观察到这样的现象：在进行设计工作或文档撰写的过程中，始终纠结在字号、颜色、排版等格式细节中，反而忽略了真正重要的内容，导致整体的工作进展缓慢。这样的现象更容易出现在完美主义者身上。

对一些文字工作者也是同理。现代计算机提供了丰富的软件帮助他们完成创作，过多的格式选择却让某些人沉溺于排出看起来完美的文字，而不是内容上完美的文字作品。这当然可以看成是技术影响文化的一种表现，但显然是不好的方面。

追究这些现象背后的原因，只能说这是由人性决定的。当一件困难的任务和一件容易的任务同时摆在面前时，正常人通常倾向于先完成容易的任务。格式的调整很明显要比内容创造简单得多。

格式化这件事究竟要在创作过程中进行，还是创作尾声进行？我本人是倾向于后者的。不可否认的是，格式与内容有某种程度上的相辅相成关系，但这依然不影响在创作尾声进行统一的格式调整。这样做还有一个优点，在进行格式调整时已经拥有了一种全局化的思维，有利于将工作做得更加出色。

由此可以想到的这两年互联网圈层出不穷的新鲜词汇，本质上这些词汇是一种格式，一种美化，一种包装。在这样一个圈子里，要时刻警惕不要陷入这些光怪陆离的概念中，而是始终关注本质内容和底层逻辑。

评《娱乐至死》
